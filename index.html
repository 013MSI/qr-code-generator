QR Code Layout

The layout consits of a 21x21 grid, which inludes 3 orientation squares, 11 boxes for color data, 3 boxes for size information, and a 7x7 logo region in the center. The rest of the grid is used to store encoded text.

Decoding Process

1. Convert the QR code image into a 21x21 grid of numbers.

The bitmap is transformed into a grid of number. The bitmap can be given in any rotation and the decoder will automaticall rotate it in the correct direction with the orientation squares.

2. Determine the encoded text's length.

3. Traverse the QR code grid, converting each 3 box sequence from the data region into the ASCII number.

4. Map each ASCII number to it's corresponding character.

Voila! The text has been sucessfully extraced from the QR code :)

Checksum

What is a checksum?

A checksum is a simple function that is used for basic error detection. While it is not fool proof, it is a quick way to do a simple check.

Our Checksum

Our program outputs a checksum when the program is generated. The user will say this and double check it when they decoded, as another checksum is printed. If the checksums are the same we can says that the data is not tampered with. Otherwise is they do not match, then the data may not be the same anymore.
